design systems

Personal definition:

A design system is a collection of patterns, designs, and carefully crafter elements that can be used together for web design.

A good design system is flexible, interchangable and easily recognized as a family of elements working together to convey a message to a user.

Popular Systems

  • Material Design
  • iOS or Apple Systems
  • Airbnb Design
  • IBM Design

Design Use

  •   Increase design effeciency
  •   consistent design output
  •   Flexibility in collaboration with team members

Developer Use

  •   Easy to grab code snippits
  •   Documentation of how to use design patterns
  •   Increased productivity
  •   Creates focus and expedites the implimentation process
  •   It makes the dev process easier

Design Elements: Building blocks of a system

Buttons, Typography, Color, Grid and use of interactions

Building and adding to a design system has greatly impacted my workflow. I can more confidently work with efficiently and accurately on multiple projects. A large part of the process was breaking down the elements into commonly used pattern Ive been really fortunate to work in many different roles in I.T. In my most recent role I created a master design file. This design file included all of the design patterns on our major applications.

In our adobe XD suite we switched from Photoshop to XD. This application is one of the major design softwares like sketch and figma. In XD the repeat grid was very useful in duplicating and modifying patterns for future use. Our team works with the foundation zurb grid. The 1200px width container was the base for the custom sizing of design elements.

How to I used design systems

  •    I built a design system for multiple applications based on brand and web standards of respected companies.
  •    The design systems I developed helped create clarity in the design decision making
  •    It helped create a consistent design across multiple mobile devices and native apps.
  •    I used the design library to established a consitent layout to the users. This helps develop trust and understanding of what may be seen next by the person using the application.
  •     It allowed me to communicate to the development team what patterns to use for the dev process.

Summary

If you are a designer and or developer, I would recommend learning about design systems. Awareness of this methodolgy will help increase you organization and speed a frontend professional.

As a designer ive found that working with design systems will teach you more about how to create flexible design elements that can work together seemlessly.
